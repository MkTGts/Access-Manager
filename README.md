# Access Manager

Локальный менеджер учетных данных на базе Python, Tkinter и SQLite.

## Основные возможности

- Организация учетных данных по разделам (например, Серверы, Облако, Офисные сервисы)
- Хранение логина, пароля и комментариев для каждой записи
- Генерирование безопасных паролей
- Поиск по всем разделам по названию или логину (без учета регистра)

## Требования

- Python 3.6+
- Tkinter (обычно входит в состав Python)
- SQLite3 (входит в состав Python)

## Установка

1. Скопируйте проект в нужную папку
2. Откройте терминал в директории проекта
3. Создайте виртуальное окружение (рекомендуется):
   ```
   python -m venv .venv
   ```
4. Активируйте виртуальное окружение:
   - Windows: `.venv\Scripts\activate`
   - Linux/Mac: `source .venv/bin/activate`
5. Запустите приложение:
   ```
   python main.py
   ```

## Руководство пользователя

### Разделы
- **Создание**: Нажмите "Добавить раздел" в левой панели
- **Просмотр описания**: Двойной щелчок по названию раздела
- **Удаление**: Выберите раздел и нажмите "Удалить раздел"

### Записи
- **Создание**: Выберите раздел, нажмите "Добавить запись" или правый щелчок → "Добавить запись"
- **Редактирование**: Двойной щелчок по записи или правый щелчок → "Редактировать запись"
- **Удаление**: Правый щелчок → "Удалить запись" или нажмите "Удалить запись"
- **Генерирование пароля**: В диалоге редактирования нажмите "Сгенерировать пароль"

### Поиск
- Выберите критерий поиска (Название или Логин)
- Введите текст для поиска
- Нажмите "Искать" 
- Поиск работает по всем разделам без учета регистра
- "Очистить" вернет вам на просмотр текущего раздела

## База данных

Приложение использует SQLite базу данных (`access_manager.db`) с двумя таблицами:
- `sections`: id, name, description
- `entries`: id, section_id, name, login, password, comment

Все поля записей являются опциональными (могут быть пустыми).
